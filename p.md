Platform: Elms,



**brief-description:**

This is a lms made with html, css, js, php, no frameworks. should be a plug n play system, user can upload the files to regular hosting and everything should work. there should be a setup wizard just like wordpress so it will be easy to setup. the lms should follow the latest design languages from apple, google and Samsung ui. 



**Admin-page:**

view, manage students in a intuitive and easy way. has a site settings section that allows admins to change core site settings. the admin dashboard should be informative but easy to handle.



**LMS-system:**

**new grades(grade 10, 11  etc), new subjects, new units can be added, edited by the admin. when students sign up they are asked which grade they are in. then the student dash should show the subjects, units available for them. if they haven't paid they should see a btn to pay.** 

**in the student dash they should see upcoming classes and the classes they paid for and a way to watch recordings if they are available. so the tutor adds a class at 8am on sat from the admin dash under a specific grade and subject. the respective students should see that class as upcoming. the tutor is going to add a title and zoom link. students can click to join right away. there should be a calendar page for the admin to add classes so he can have a clear schedule. class recordings are added as youtube links and students can view the videos from the site without going to yt under the subjects they paid for.**



**Payment system:** 

**student signs up for his grade. sees the subjects for his grade. has pay btn. takes to page. this page have image with instructions on how to pay (this image can be setup from admin core site settings) and a btn to upload payment proof as image. the uploaded payment slips/images are viewable from the admin side. admin see student name, grade, subject he paid, and proof image(expandable). admin can approve payment or if theres a problem contact the student. when approved the videos and classes under that subject will be unlocked for the student.** 



**nortification system: basically a insite norti system to let admin see new payments, let admin remember his upcoming classes, same for students.**



**store-page: the page where students can browse subjects and select. not registered students can  see/ select grade>subject>unit then will show any videos the tutor uploaded as lesson pack. unregistered students will not see upcoming classes or recordings. for registered users they will see everything and should have a way to filter lesson packs, upcoming, recordings.**



**home-page: a homepage with attractive text and elements. should be professional. should link to register and browse pages.**



**admin-login: hardcoded admin login. usrname:admin pswd:admin. sitelink/admin <--admin page.**
